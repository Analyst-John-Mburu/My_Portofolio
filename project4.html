Project 4

E-COMMERCE ANALYSIS

The E-commerce Data Modelling project focused on designing and implementing a structured database schema to store and manage transactional and customer data efficiently. Using SQL for modeling and Python for analysis, the project aimed to uncover valuable insights into customer behavior and optimize sales strategies to improve overall business performance.

Objective

  • Design a relational database schema to effectively store e-commerce data.
  • Extract and transform data using SQL queries for efficient analysis.
  • Analyze customer behavior and purchasing patterns using Python.
  • Provide insights to support marketing strategies and business decisions.

Key Insights

  • Customer Segmentation: Repeat customers had significantly higher average order values, highlighting the importance of loyalty programs.
  • Sales Trends: Peak sales occurred during weekends and promotional periods, suggesting opportunities for targeted campaigns.
  • Product Performance: Certain categories consistently outperformed others, guiding inventory and marketing focus.
  • Cart Abandonment: A noticeable drop-off was found during checkout by mobile users, indicating UX improvement needs.
  • Geographic Insights: Urban customers generated more revenue per transaction than rural ones, aiding in regional targeting.
